Chatterbots, Panda and ELIZA

Curtis Thompson Thursday 31st March 2016 Artificial Intelligence

Yesterday I thought it would be a fun idea to create a computer program which I could have a conversation with. These types of programs are known as chatterbots and creating one as intelligent as the famous Cleverbot or A.L.I.C.E would be an impossible task for one days work. The short time limit led me to look back at one simple program from the 1960's, ELIZA.

ELIZA is one of the first chatterbots, created by Joseph Weizenbaum between 1964 and 1966. It is a very simple expert system chatterbot that searches for keywords and then gets phrases from the user's responses (depending on the keyword), combining these with its own pre-programmed phrases so it can reply. This sounded easy, and looked even easier when I found a 260 line source code in BASIC for an early version of ELIZA. I therefore decided to write my own version of ELIZA based on this source code and other information about ELIZA.

A conversation between a user and ELIZA.

To write my chatterbot I would have to write a few small functions:

  • Finding keywords in user's responses
  • Getting phrases out of user's responses
  • Replacing pronouns (and other words) in phrases
  • Choosing and combining phrases to make the chatterbot reply

My chatterbot, like ELIZA, does not learn new keywords and phrases over time so these must all be programmed into my chatterbot from the start. A majority of keywords I chose were similar to keywords used by ELIZA or are simple keywords that people often use in everyday speech such as "I have", "yes" and "think". Along with each of these keywords I had to write a few possible phrases the chatterbot could use in reply to them and denote whether these phrases should be combined with phrases gathered from the user.

Once the keywords and phrases were chosen I could start programming. I chose C# for this simply because I hadn't used C# for a while, this program would be simple enough that I didn't need to care about which programming language to use. The program had to get the user's input, search for keywords with IF... ELSE IF... statements, find and combine phrases, and then output the chatterbots reply. This would be repeated continuously.

A flowchart showing how the chatterbot decides what to say.

Once this was done I had a working chatterbot! However there were two main problems I had not yet addressed; pronouns not being replaced and how to handle user responses with no keywords. Writing the function for pronoun replacement was simple as I could use the same replacements that ELIZA used, looping through these replacements to make the phrases make sense. Handling responses with no keywords was also not too difficult. If the response had a question mark the chatterbot could prompt for a different question or topic, otherwise it would use very short generic replies such as "yeah" or "ummmm...". With this completed the chatterbot was essentially complete. I called it Panda!

Simple chatterbots like Panda and ELIZA do not learn, they can only work with what they are given at the start. Besides giving it the ability to learn there are many possible improvements to the program. One would be to have more keywords and phrases that it can search for and use, making the conversations more in depth or complex. Another would be giving the program a 'memory', so that if the user provides some information the program can continue to refer back to that piece of information. A third idea would be to improve the searching algorithms so that the program can create better responses. Like most AI programs, the potential improvements are endless.

A conversation with Panda, the completed chatterbot.

This is just one of the ways of creating a chatterbot, and one of the quickest I could find. The possibilities are nearly endless with this. If you want to see the C# code I used for Panda, it can all be found in this repository. If you want more information on ELIZA, this paper by Joseph Weizenbaum will help. And finally, click here to see the project. I'd love to see people create their own, more advanced, versions.
