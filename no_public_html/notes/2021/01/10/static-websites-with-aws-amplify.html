Static Websites with AWS Amplify Curtis Thompson Sunday 10th January 2021 Projects and Programming

Static websites offer several advantages to dynamic websites, and converting a dynamic website to a static one should be something to consider if possible. Since a static website is built only from HTML, CSS, and Javascript it is easy for someone to build. Since the server will only be returning file to users, page-load times are also much quicker, and costs lower as there is no need for expensive backend servers.

That's why I recently considered changing my personal site from a PHP filled dynamic website, to a simple HTML/CSS/Javascript static website. I did this using just:

  • AWS Amplify, for static web hosting
  • Amazon Route 53, for domain management
  • GitHub, for updating your website and version control

Let's have a quick look at how it's done.

Get Your Code On GitHub

Once you have coded up your static pages, the first step is to create a repository on GitHub, upload them to there. I won't explain the details of how to do this as it should be familiar to anyone coding their own static website. If you are struggling at this step though, just sign up to GitHub and they will have step-by-step instructions on how to do it - it shouldn't take ant longer than 5 minutes.

Connect Your Repository To AWS Amplify

The next step is to connect your new GitHub repository to the AWS Amplify console. To do this, go on the AWS Amplify homepage. You should find two options, either "Develop" or "Deliver". You want to select the deliver option to host your web app, which is a static website in this case.

Select the "Deliver" option to host a static website.

You will then be given the option to connect to GitHub, which is what you want to do. When connected, you will be able to select one of your repositories, and the specific branch of that repository. If you are unsure about what the branch settings are, just leave it as the default given by AWS.

Next, you will go through build settings. For now, we can leave that as it is. For my personal website I used this to select a specific directory in my repository, but for this tutorial we will leave it for now. Review all of your settings, and then save and deploy.

How your console should look when AWS Amplify is deploying your website.

Your website is now being deployed, and there will be a link to view it once deployment is complete.

Get Your Domain With Amazon Route 53

Amazon Route 53 is an AWS service that allows you to manage domain names. If you want a domain for your static website, you can buy one here. You can also transfer a domain name over from another domain management service, which is what I did.

Once you have your domain name set up, you can connect it to your website in the AWS Amplify settings, under "Domain Management".

Updating Your Website

At this point, you don't need to ever view AWS Amplify again, if you don't want to.

If you want to update your website, push the updates straight to your GitHub repository. AWS Amplify will then detect that there has been an update, and will update your website for you. And that's all!
