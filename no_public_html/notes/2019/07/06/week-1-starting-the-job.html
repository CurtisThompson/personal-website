Week 1: Starting The Job Curtis Thompson Saturday 6th July 2019 Personal

This week I started with Warwick Students' Union as a Website Content Assurance Executive. I'll only be working with them for a few weeks as they finalise their new website, as I check the site to make sure everything is ready to go live.

Checking a site is ready to go live, what does this mean? I'll be checking that all scripts are working, all links lead to other pages (as opposed to 404ing), and all pages match the agreed designs. Obviously there is a lot more to check, there are so many different things that I possibly couldn't list them all now. However, it is essentially my job to make sure everything is perfect. And that is what I have already started doing.

One of my first tasks (once I got through all of the HR induction stuff) was to make a particular page look like the mockup design. As the template had been coded before I began the job, it wasn't too difficult. The main issue was the side navigation bar, as several buttons were not displaying while others were directing users to the live site (as opposed to the dev site that we were working on). Directing to the live site is a problem as directories of the new site have changed so when the new site goes live, all of these links will 404. This is why it is good to use relative links as opposed to absolute links, and why I need to fix it.

I explored the buttons, and found they were using absolute redirects. This idea led me to find bigger issues. I checked other redirects on the website, and they were absolute too - leading to the live site. Big issue! So I spent a few hours going through the redirects, trying to fix them. Overall I found that 65% of the redirects needed to be changed, and there was also 7% that couldn't be changed due to the restructuring of the new site - they had to be deleted or flagged up in a report. Most importantly, the navigation bar was fixed.

As my first week progressed, I moved onto different jobs. I had been given some deadlines, and had to prioritise my work to ensure everything could get done. These tasks included restructuring sections, applying styling, and quality checking pages. Although the site was mostly fine, there were a few recurring issues:

  • Headings (h1, h2, h3, ...) had no consistency in their application
  • Paragraphs (p) were being used to create lists, as opposed to the list tags (ul, ol, li)
  • Images were not resizing when the device or browser resized, mostly causing issues on mobile

Issues that I couldn't fix were flagged up in reports. I found it fun to fix bugs, so would try to do so myself if it wouldn't take hours to figure it out. Obviously those that I had listed above were ones I could fix, but there were some more complex scripting issues that I decided to flag up so that the main web developer could fix them instead. Next week I will be doing more coding myself, but it was a good first week that allowed me to get used to the system and my job.
